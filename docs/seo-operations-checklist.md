## 運用チェックリスト

### UX改善前

- TOPで `何をすればいいか` が1画面で分かるか確認する
- 同じ役割の導線が上部に重なっていないか確認する
- 検索前と検索後で、主導線がぶれていないか確認する

### 記事追加前

- 同地域で `入口 / ジャンル / 子ども向け / 目的別` のどこが足りないか確認する
- 既存記事と `title / description / H1` が被らない切り口を決める
- 記事で紹介する教室が一覧データにあるか確認する

### 記事作成時

- 新記事に `地域入口 / 年齢別 / 子ども向け / 体験 / 料金` の関連記事を入れる
- 金額を本文で出したら `data/studios.js` の `pricing.minPrice` に反映できるか確認する
- FAQは `地域名 + 年齢 + 比較軸` を意識して書く
- 構造化データの `headline / description / breadcrumb` が本文とずれていないか確認する

### 記事公開時

- `recommendations/index.html` の件数とカードを更新する
- 地域記事を増やしたら `recommendations/ehime-local-lessons/index.html` にも反映する
- URL追加時は `sitemap.xml` を更新する
- 必要に応じて関連地域記事の内部リンクも更新する

### 本番反映前

- トップページを確認する
- 特集一覧ページを確認する
- 新規記事を確認する
- リンク切れと表示崩れがないか最低限見る
- TOPで `入口 -> 検索 -> 結果` の流れが自然か確認する
- 検索結果で表示要素が多すぎないか確認する
- 比較メモとお気に入りの違いがUI上で伝わるか確認する

### UI確認 5分版

- TOPヒーローで `一覧ですぐ比較する` と `入口から探し方を選ぶ` のどちらを押せばいいか迷わないか確認する
- TOPの `最初の1歩だけ選ぶ` 4枚が、1画面内で見渡しやすいか確認する
- 検索結果カードで `対象 / 料金 / 通いやすさ` が最初に目に入るか確認する
- 検索結果上部の `合わせて見たい特集` が多すぎず、補助情報として読めるか確認する
- `recommendations` のヒーロー、START HERE、記事カードの順に視線が流れるか確認する
- `recommendations` の記事カードで、タイトルと説明が詰まりすぎて見えないものがないか確認する
