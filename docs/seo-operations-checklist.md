## 運用チェックリスト

### 関連記事運用ルール

- 市町別の個別記事は、まず `同じ市町の記事` を優先して関連記事に入れる
- 同じ市町で足りないときは、`愛媛の子ども向け / 年齢別 / 体験 / 料金 / 地域別` の県ハブへ広げる
- `松山の個別記事 -> 他市町の個別記事` のような直接導線は基本つくらない
- 他エリアの記事をまたぐ導線は、`愛媛の地域別習い事ガイド` や `特集一覧` のようなハブページ側で持つ
- 関連記事は `同エリア -> 同目的の県ハブ -> 一覧検索` の順で考える
- 主力ページへ戻したいときも、ユーザー文脈に合う導線だけ残し、関連性が薄いリンクは入れない
- 関連記事ラベルは、`おすすめ◯選` より `比較ガイド / 入口ガイド / 習い事ガイド` を優先して、本文の役割とトーンを合わせる

### UIでは削る / SEOでは残す ルール

- TOP、一覧ページ、地域ハブの `ヒーロー / 導入文 / カード説明` は短くしてよい
- SEOで必要な説明は、`このページが向いている人 / 比較軸 / FAQ / 記事末まとめ` に集める
- カードは `タイトル + 短い一言 + 次の行動` に寄せ、長文は本文に持たせる
- `見せる文字` を減らしても、`H2/H3`, FAQ, 内部リンク, 比較軸の本文は削りすぎない
- 特に主力記事では、`向いている人`, `比較軸`, `地域性`, `FAQ` を残して評価を支える
- 文章を削るときは、`導入文` から先に削り、`検索意図に答える本文` は後ろに残す
- UI改善で文章を減らしたら、`meta description`, `H1`, FAQ, 構造化データとの整合性も確認する

### UX改善前

- TOPで `何をすればいいか` が1画面で分かるか確認する
- 同じ役割の導線が上部に重なっていないか確認する
- 検索前と検索後で、主導線がぶれていないか確認する

### 記事追加前

- 同地域で `入口 / ジャンル / 子ども向け / 目的別` のどこが足りないか確認する
- 既存記事と `title / description / H1` が被らない切り口を決める
- 記事で紹介する教室が一覧データにあるか確認する

### 記事作成時

- 新記事に `地域入口 / 年齢別 / 子ども向け / 体験 / 料金` の関連記事を入れる
- 個別記事の関連記事は `同エリア優先` になっているか確認する
- 金額を本文で出したら `data/studios.js` の `pricing.minPrice` に反映できるか確認する
- FAQは `地域名 + 年齢 + 比較軸` を意識して書く
- 構造化データの `headline / description / breadcrumb` が本文とずれていないか確認する
- 公式ページがトップへリダイレクトしていないか、スクール一覧や教室ページで現行稼働が読めるかも確認する
