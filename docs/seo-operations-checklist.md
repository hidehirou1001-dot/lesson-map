## 運用チェックリスト

### 記事追加前

- 同地域で `入口 / ジャンル / 子ども向け / 目的別` のどこが足りないか確認する
- 既存記事と `title / description / H1` が被らない切り口を決める
- 記事で紹介する教室が一覧データにあるか確認する

### 記事作成時

- 新記事に `地域入口 / 年齢別 / 子ども向け / 体験 / 料金` の関連記事を入れる
- 金額を本文で出したら `data/studios.js` の `pricing.minPrice` に反映できるか確認する
- FAQは `地域名 + 年齢 + 比較軸` を意識して書く
- 構造化データの `headline / description / breadcrumb` が本文とずれていないか確認する
- 複数店舗がある系列は `チェーン単位` ではなく `掲載店舗名` を明記し、必要なら `data/studios.js` に掲載方針メモを入れる

### 記事公開時

- `recommendations/index.html` の件数とカードを更新する
- 地域記事を増やしたら `recommendations/ehime-local-lessons/index.html` にも反映する
- URL追加時は `sitemap.xml` を更新する
- 必要に応じて関連地域記事の内部リンクも更新する

### 本番反映前

- トップページを確認する
- 特集一覧ページを確認する
- 新規記事を確認する
- リンク切れと表示崩れがないか最低限見る

## 簡易QAチェックリスト

### 新記事を追加した日

- `title / description / H1` が同系統の記事と被っていない
- `FAQ` の見出し文言と `FAQPage` の質問内容が一致している
- `Article` or `CollectionPage` の `headline / description / dateModified` が本文とずれていない
- 関連記事に `地域 / 年齢 / 子ども向け / 体験 / 料金` のどれかが入っている
- 記事内で触れた教室が `data/studios.js` に存在する
- チェーン系の教室やジムは `系列名だけ` ではなく `掲載店舗名` が本文か表に入っている

### 本番反映前の5分確認

- トップの検索結果が表示される
- 比較メモ、お気に入り、共有導線が崩れていない
- `recommendations/index.html` の件数とカードが更新内容に合っている
- 新規追加 or 更新した記事の `関連記事` が意図どおりにつながる
- `sitemap.xml` に追加URLと `lastmod` が反映されている

### 公開直後の確認

- スマホ幅でトップと更新記事の見出しが崩れていない
- FAQの開閉が動く
- 外部リンクが明らかに切れていない
- 共有ボタンでURLが崩れていない
- 更新日や確認状況の表示が抜けていない

### 月次運用

- 料金、体験、リンク切れ、更新日の見直しを行う
- `料金不明` の教室を優先的に補完する
